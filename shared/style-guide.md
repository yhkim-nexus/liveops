# 스타일 가이드

모든 에이전트가 따라야 하는 문서 작성 스타일입니다.

## 1. 문서 구조

### 1.1 YAML Front Matter (필수)

모든 산출물 파일 최상단에 메타데이터 포함:

```yaml
---
id: "PRD-001"
title: "문서 제목"
project: "프로젝트명"
version: "v1.0"
status: "draft | review | approved | archived"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
author: "agent-name"
reviewers: []
related_docs:
  - "RES-001"
  - "MTG-001"
tags:
  - "태그1"
  - "태그2"
---
```

### 1.2 문서 헤더

```markdown
# [문서 타입]: [제목]

> 한 줄 요약 (Optional)

## 문서 정보

| 항목 | 내용 |
|------|------|
| ID | [ID] |
| 버전 | [버전] |
| 상태 | [상태] |
| 작성일 | [날짜] |
| 작성자 | [에이전트] |
```

## 2. 작성 원칙

### 2.1 명확성
- 모호한 표현 금지: "적절한", "빠른", "좋은" → 구체적 수치/기준
- 한 문장 = 하나의 의미
- 전문 용어는 용어집 참조 링크

### 2.2 일관성
- 용어: `.claude/shared/terminology.md` 준수
- 네이밍: `.claude/shared/conventions.md` 준수
- 포맷: 이 스타일 가이드 준수

### 2.3 추적성
- 모든 항목에 고유 ID 부여
- 관련 문서 상호 참조
- 변경 이력 기록

## 3. 포맷팅

### 3.1 제목 레벨
```
# H1: 문서 제목 (1개만)
## H2: 주요 섹션
### H3: 하위 섹션
#### H4: 상세 항목
```

### 3.2 목록
- 순서 없는 목록: `-` 사용
- 순서 있는 목록: `1.` 사용 (자동 번호)
- 중첩은 2단계까지만

### 3.3 표
```markdown
| 헤더1 | 헤더2 | 헤더3 |
|-------|-------|-------|
| 내용 | 내용 | 내용 |
```

### 3.4 코드
- 인라인: `코드`
- 블록: 언어 명시
```yaml
key: value
```

### 3.5 강조
- **굵게**: 중요 키워드
- *기울임*: 용어, 강조
- `코드`: 기술 용어, 파일명

## 4. 링크 규칙

### 4.1 내부 문서 링크
```markdown
관련 문서: [[PRD-001]] 참조
```

### 4.2 외부 링크
```markdown
[링크 텍스트](URL)
```

### 4.3 섹션 링크
```markdown
[섹션명](#섹션-앵커)
```

## 5. 상태 정의

| 상태 | 설명 | 다음 단계 |
|------|------|----------|
| `draft` | 초안 작성 중 | review |
| `review` | 검토 중 | approved / draft |
| `approved` | 승인 완료 | archived |
| `archived` | 보관 (더 이상 수정 안 함) | - |

## 6. 버전 규칙

- `v1.0`: 최초 완성
- `v1.1`: 소규모 수정
- `v2.0`: 대규모 변경

변경 시 문서 하단 변경 이력 업데이트 필수.
