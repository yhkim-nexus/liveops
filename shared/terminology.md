# 용어집 (Terminology)

프로젝트 전체에서 일관되게 사용하는 용어 정의입니다.
모든 에이전트는 이 용어집을 따릅니다.

## 사용자 관련

| 용어 | 정의 | 사용 예 | 금지 표현 |
|------|------|---------|----------|
| 회원 | 서비스에 가입한 사용자 | "회원 가입", "회원 정보" | 유저, 사용자, 고객 |
| 비회원 | 가입하지 않은 방문자 | "비회원 구매" | 게스트, 손님 |
| 관리자 | 서비스 운영 권한자 | "관리자 페이지" | 어드민, 운영자 |

## 기능 관련

| 용어 | 정의 | 사용 예 | 금지 표현 |
|------|------|---------|----------|
| 로그인 | 계정 인증 및 세션 생성 | "로그인 화면" | 사인인, 접속 |
| 로그아웃 | 세션 종료 | "로그아웃 버튼" | 사인아웃, 접속 해제 |
| 회원가입 | 계정 생성 | "회원가입 절차" | 가입, 등록 |
| 탈퇴 | 계정 삭제 | "회원 탈퇴" | 삭제, 해지 |

## 상태 관련

| 용어 | 정의 | 사용 예 |
|------|------|---------|
| 활성 | 정상 사용 가능 상태 | "활성 상태" |
| 비활성 | 일시 중지 상태 | "비활성 처리" |
| 정지 | 제재로 인한 사용 불가 | "이용 정지" |
| 삭제 | 영구 제거 | "계정 삭제" |

## 결제 관련

| 용어 | 정의 | 사용 예 |
|------|------|---------|
| 결제 | 대금 지불 | "결제 완료" |
| 환불 | 결제 금액 반환 | "환불 요청" |
| 구독 | 정기 결제 | "구독 서비스" |
| 해지 | 구독 종료 | "구독 해지" |

## 문서 관련

| 용어 | 정의 |
|------|------|
| PRD | Product Requirements Document, 제품 요구사항 문서 |
| 화면정의서 | UI/UX 스펙 문서 |
| 운영정책 | 서비스 운영 규칙 문서 |
| 플로우차트 | 프로세스 흐름도 |

## 에이전트 관련

| 용어 | 정의 |
|------|------|
| planner | 기획 총괄 에이전트 |
| researcher | 리서치 에이전트 |
| prd | PRD 작성 에이전트 |
| policy | 정책 작성 에이전트 |
| uiux-spec | 화면정의서 작성 에이전트 |
| diagram | 다이어그램 작성 에이전트 |
| reviewer | 검토 에이전트 |
| meeting-note | 회의록 작성 에이전트 |

## 상태값 (영문)

| 상태 | 의미 |
|------|------|
| draft | 초안 |
| review | 검토 중 |
| approved | 승인됨 |
| archived | 보관됨 |
| pending | 대기 중 |
| in_progress | 진행 중 |
| completed | 완료 |
| blocked | 차단됨 |

---

## 용어 추가 규칙

1. 새 용어 추가 시 이 문서 업데이트
2. 동의어는 하나로 통일
3. 영문 용어는 한글 병기 고려
4. 약어 사용 시 첫 등장에서 풀네임 표기
