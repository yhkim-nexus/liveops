# 리뷰 체크리스트

reviewer 에이전트가 각 산출물을 평가할 때 사용하는 체크리스트입니다.

## 1. 공통 체크리스트

모든 문서에 적용:

### 1.1 메타데이터
- [ ] YAML front matter 존재
- [ ] 필수 필드 모두 작성 (id, title, version, status, created, author)
- [ ] 버전 형식 올바름 (v1.0)
- [ ] 상태값 유효함 (draft/review/approved/archived)
- [ ] 관련 문서 참조 정확함

### 1.2 형식
- [ ] 스타일 가이드 준수 (.claude/shared/style-guide.md)
- [ ] 용어집 용어 사용 (.claude/shared/terminology.md)
- [ ] 네이밍 컨벤션 준수 (.claude/shared/conventions.md)
- [ ] 제목 레벨 구조 올바름
- [ ] 표, 목록 형식 일관됨

### 1.3 완결성
- [ ] 모든 필수 섹션 존재
- [ ] 빈 섹션 없음 (또는 N/A 명시)
- [ ] 변경 이력 작성됨

---

## 2. 리서치 리포트 (RES) 체크리스트

### 2.1 내용
- [ ] Executive Summary 존재
- [ ] 출처 모두 명시
- [ ] 데이터 날짜/시점 명시
- [ ] 사실과 의견 구분

### 2.2 분석
- [ ] 시장 규모 데이터 포함
- [ ] 경쟁사 최소 3개 분석
- [ ] SWOT 또는 유사 프레임워크 사용
- [ ] 실행 가능한 인사이트 제공

### 2.3 품질
- [ ] 정량적 데이터 포함
- [ ] 최신 정보 (6개월 이내)
- [ ] 신뢰할 수 있는 출처

---

## 3. PRD 체크리스트

### 3.1 구조
- [ ] Overview 섹션 존재
- [ ] Target Users 정의됨
- [ ] Features & Requirements 존재
- [ ] Non-Functional Requirements 존재
- [ ] Dependencies & Constraints 명시
- [ ] Release Plan 존재

### 3.2 기능 명세
- [ ] 모든 기능에 고유 ID (F-001)
- [ ] 우선순위 명시 (P0/P1/P2)
- [ ] 사용자 스토리 형식 사용
- [ ] Acceptance Criteria 존재 (Given-When-Then)

### 3.3 품질
- [ ] 모호한 표현 없음
- [ ] 측정 가능한 성공 지표
- [ ] 예외 상황 정의됨
- [ ] 리서치 결과 반영됨

### 3.4 추적성
- [ ] 회의록 참조됨
- [ ] 리서치 참조됨
- [ ] 요구사항 ID 체계 일관됨

## 8. 평가 기준

| 등급 | 기준 |
|------|------|
| ✅ Pass | 필수 항목 100%, 권장 항목 80% 이상 |
| ⚠️ Conditional | 필수 항목 100%, 권장 항목 60-80% |
| ❌ Fail | 필수 항목 미충족 |

### 심각도 분류

| 심각도 | 정의 | 조치 |
|--------|------|------|
| 🔴 Critical | 서비스 영향, 법적 문제 | 즉시 수정 필수 |
| 🟡 Major | 품질 저하, 혼란 유발 | 승인 전 수정 |
| 🟢 Minor | 개선 권장 | 다음 버전에 반영 |
