## 1. 프로젝트 개요

### 1.1 프로젝트 목적

게임 라이브옵스(LiveOps) 시장을 분석하고, 이를 기반으로 게임 운영에 필요한 라이브옵스 서비스를 기획한다. 라이브옵스는 게임 출시 이후 지속적으로 콘텐츠를 업데이트하고, 이벤트를 운영하며, 사용자 참여를 유도하여 게임의 수명과 수익을 극대화하는 운영 전략이다.

### 1.2 프로젝트 배경

- 게임 산업에서 라이브옵스는 GaaS(Game as a Service) 모델의 핵심 전략으로 자리잡았다
- 게임 출시 후 운영 단계에서의 사용자 유지(retention)와 수익화(monetization)가 게임 성공의 핵심 요인이다
- 체계적인 라이브옵스 서비스를 통해 게임사의 운영 효율성과 사용자 경험을 동시에 개선할 수 있다

### 1.3 프로젝트 범위

| 항목 | 포함 | 제외 |
|------|------|------|
| 시장 분석 | 국내외 라이브옵스 서비스 시장 | 게임 개발 도구 시장 |
| 기능 분석 | 이벤트 관리, 푸시 알림, A/B 테스트, 인앱 메시징, 리모트 컨피그, 분석/대시보드 | 게임 엔진, 서버 인프라 |
| 사용자 분석 | 게임사 운영팀, PM, 마케터 | 최종 게임 플레이어 |

## 2. 1차 목표: 시장조사

### 2.1 조사 목적

게임 라이브옵스 시장의 현황을 파악하고, 주요 서비스들의 기능을 비교 분석하여 기획의 기초 자료를 확보한다.

### 2.2 조사 범위

1. **현재 운영 중인 서비스 파악**
   - 글로벌 라이브옵스 플랫폼 (예: Unity LiveOps, PlayFab, GameAnalytics, Leanplum 등)
   - 국내 라이브옵스 서비스 및 솔루션
   - 게임사 자체 구축 사례

2. **기능 비교**
   - 이벤트 관리 (스케줄링, 타겟팅, 보상 시스템)
   - 리모트 컨피그 (실시간 설정 변경, 세그먼트별 설정)
   - A/B 테스트 (실험 설계, 결과 분석)
   - 푸시 알림 / 인앱 메시징
   - 분석 및 대시보드 (KPI 모니터링, 코호트 분석)
   - 사용자 세그먼테이션
   - 오퍼/프로모션 관리

3. **장단점 분석**
   - 각 서비스의 강점과 약점
   - 가격 정책 및 비용 구조
   - 통합 용이성 (SDK, API)
   - 사용자 경험 (관리자 UI/UX)
   - 확장성 및 커스터마이징 가능 여부

## 3. 다음 단계

### 3.1 researcher 에이전트 시장조사 위임

리서치 단계를 위해 researcher 에이전트를 spawn하여 다음 작업을 병렬로 수행한다:

1. **market-researcher**: 게임 라이브옵스 시장 전체 분석
   - 시장 규모 및 성장 추세
   - 주요 트렌드 (GaaS, 실시간 운영, 데이터 기반 의사결정)
   - 기술 트렌드 (AI 기반 개인화, 실시간 분석 등)

2. **competitor-researcher**: 주요 경쟁 서비스 심층 분석
   - Unity LiveOps, Microsoft PlayFab, AWS GameSparks
   - GameAnalytics, deltaDNA, Leanplum
   - 국내 서비스 및 자체 구축 사례

3. **user-researcher**: 타겟 사용자(게임사 운영팀) 니즈 분석
   - 게임사 규모별 니즈 (인디/중소/대형)
   - 장르별 라이브옵스 요구사항
   - 현재 pain point 및 unmet needs
