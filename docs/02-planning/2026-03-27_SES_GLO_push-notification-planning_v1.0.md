# 기획 세션: 모바일 푸시 알림 발송

> LiveOps 관리 콘솔에서 모바일 푸시 알림을 직접 작성·발송·분석하는 기능의 기획 산출물(PRD, UX 스펙, 다이어그램)을 생성하는 계획을 정의한다. 운영팀이 캠페인성 푸시 메시지를 타겟 오디언스에게 발송하고, 발송 결과를 분석할 수 있는 전체 워크플로우를 포함한다.

## 1. 프로젝트 개요

### 1.1 목적

운영팀이 모바일 푸시 알림을 직접 작성·발송·분석할 수 있는 캠페인 관리 기능을 정의한다. 기존 F-016(웹훅/알림)은 이벤트 기반 내부 알림이었으나, 이번 기능은 운영팀이 직접 캠페인성 푸시 메시지를 타겟 오디언스에 발송하는 것이 목적이다. 메시지 작성, 오디언스 타겟팅, 스케줄링, 템플릿 관리, 발송 분석, 빈도 제한, A/B 테스트까지 푸시 알림 운영 전체 라이프사이클을 다룬다.

### 1.2 배경

- SES-GLO-001 킥오프에서 "푸시 알림 / 인앱 메시징"이 기획 범위에 포함됨
- RES-GLO-002 경쟁사 분석에서 5/7 플랫폼이 푸시 알림 발송 기능 지원
- PRD-GLO-002 F-016에서 기초적인 웹훅/알림 기능 정의 — 이를 캠페인 발송으로 확장
- 기존 세그멘테이션 시스템(PRD-GLO-001 F-006)의 오디언스 타겟팅을 재사용하여 정교한 발송 대상 설정 가능
- PRD-GLO-003 A/B 테스트 실험 프레임워크와 연동하여 푸시 메시지 최적화 지원

### 1.3 아키텍처

Next.js 기반 관리 콘솔에서 멀티스텝 위저드를 통해 캠페인을 생성하고, FCM(Firebase Cloud Messaging)/APNs(Apple Push Notification service)를 통해 푸시 알림을 발송한다. 오디언스 타겟팅은 PRD-GLO-001 F-006 세그멘테이션 시스템을 재사용하며, 발송 스케줄링은 서버 사이드 잡 큐를 통해 처리한다. 2단계 승인 워크플로우(PRD-GLO-002 패턴)로 오발송을 방지한다.

### 1.4 기술 스택

| 영역 | 기술 | 용도 |
|------|------|------|
| 캠페인 UI | Shadcn UI (Card, Table, Button, Dialog) | 캠페인 목록, 상세, 생성 위저드 |
| 멀티스텝 위저드 | Shadcn UI (Stepper, Form) | 캠페인 생성 단계별 폼 |
| 메시지 미리보기 | Custom Component (iOS/Android 목업) | 푸시 메시지 디바이스별 미리보기 |
| 오디언스 피커 | PRD-GLO-001 F-006 오디언스 컴포넌트 재사용 | 타겟 오디언스 선택 |
| 스케줄링 | React Hook Form + Zod | 발송 시간/반복 규칙 입력 및 검증 |
| 분석 차트 | Recharts | 발송/도달/오픈/전환 퍼널, 시계열 차트 |
| 데이터 패칭 | React Query | 캠페인 CRUD, 분석 데이터 API 호출 |
| 권한 제어 | useAuth + RBAC | Operator 이상 발송, Editor 이상 초안 |

### 4.2 Step 1: PRD 작성 (prd 에이전트)

**에이전트:** prd (Sonnet)
**산출물:** `docs/03-prd/2026-03-27_PRD_GLO_push-notification_v1.0.md`

푸시 알림 발송 7개 기능(F-037~F-043)의 요구사항을 정의한다. PRD-GLO-002(라이브 이벤트) 구조를 따르며, 기존 시스템과의 연동 지점(세그멘테이션, A/B 테스트, RBAC)을 명시한다.

**기능 범위:**
1. **F-037: 푸시 메시지 작성** — 제목(50자), 본문(200자), 딥링크 URL, 이미지(선택), 커스텀 데이터(JSON), iOS/Android 미리보기
2. **F-038: 타겟 오디언스 설정** — PRD-GLO-001 F-006 오디언스 재사용, 푸시 수신 동의 필터, 플랫폼(iOS/Android) 필터, 예상 도달 수 표시, 전체/특정 오디언스/개별 선택
3. **F-039: 발송 스케줄링** — 즉시 발송, 예약 발송(날짜/시간+타임존), 반복 발송(RFC 5545 rrule), 플레이어 로컬 타임존 기반 발송 옵션, 2단계 승인 워크플로우(PRD-GLO-002 패턴 재사용)
4. **F-040: 템플릿 관리** — CRUD, 변수 지원(player_name, event_name 등 플레이스홀더), 카테고리/태그 분류
5. **F-041: 발송 분석 대시보드** — 발송/도달/오픈/전환/실패 퍼널, 시계열 차트, 오디언스별 비교 분석, CSV/Excel 내보내기
6. **F-042: 빈도 제한 및 발송 보호** — 플레이어당 일일 최대 발송 수(기본 3회, 1~10회 설정 가능), 야간 방해 금지는 OS 위임(iOS Focus / Android DND), 긴급 알림(critical)은 빈도 제한 무시
7. **F-043: 푸시 A/B 테스트** — 변형 메시지 생성(최대 4개), 트래픽 분배 비율 설정, 오픈율/전환율 기반 자동 승자 선정, PRD-GLO-003 실험 프레임워크 연동

**릴리즈 계획:**
- MVP: F-037, F-038, F-039, F-042 (메시지 작성 + 타겟팅 + 스케줄링 + 빈도 제한)
- v1.1: F-040, F-041 (템플릿 + 분석)
- v1.2: F-043 (A/B 테스트)

**검증:** PRD-GLO-001(세그멘테이션) 오디언스 모델과의 정합성, PRD-GLO-002(라이브 이벤트) 승인 워크플로우 패턴 일관성, PRD-GLO-005(RBAC) 권한 모델과의 일관성

### 4.3 Step 2: UX 스펙 작성 (uiux-spec 에이전트) — Step 1 완료 후

**에이전트:** uiux-spec (Sonnet)
**산출물:** `docs/05-ux/2026-03-27_UX_GLO_push-notification_v1.0.md`

푸시 알림 관리 화면 정의서를 작성한다. UX-GLO-003(A/B 테스트) 위저드 패턴과 UX-GLO-001(세그멘테이션) 오디언스 피커를 재사용한다.

**화면 정의서 구조:**
1. **SCR-028: 캠페인 목록** (`/push/campaigns`) — 캠페인 테이블(상태 뱃지, 발송일, 도달 수, 오픈율), 필터(상태/기간/오디언스), 검색, 페이지네이션
2. **SCR-029: 캠페인 생성/편집** (`/push/campaigns/new`) — 멀티스텝 위저드(메시지 작성 > 오디언스 설정 > 스케줄링 > 확인/승인), iOS/Android 미리보기 패널
3. **SCR-030: 캠페인 상세/분석** (`/push/campaigns/[id]`) — 캠페인 정보 요약, 발송 퍼널 차트, 시계열 차트, 오디언스별 분석, A/B 테스트 결과(해당 시)
4. **SCR-031: 템플릿 목록** (`/push/templates`) — 템플릿 카드 그리드/리스트 뷰, 카테고리 필터, 검색
5. **SCR-032: 템플릿 편집** (`/push/templates/[id]`) — 변수 삽입 에디터, iOS/Android 미리보기, 카테고리/태그 설정

**검증:** 모든 화면에 Default/Empty/Loading/Error 상태 정의, 인터랙션 ID 연속성, UX-GLO-003 위저드 패턴과의 일관성

**다이어그램 목록:**
- DIA-029: 푸시 데이터 모델 (ERD) — push_campaign, push_template, push_delivery_log, push_ab_variant 엔티티 및 관계
- DIA-030: 캠페인 라이프사이클 (상태 다이어그램) — draft > pending_approval > approved > scheduled > sending > sent > completed / failed / cancelled 전이
- DIA-031: 푸시 발송 시퀀스 (시퀀스 다이어그램) — 관리자 > 캠페인 생성 > 승인 요청 > 승인자 확인 > 스케줄러 > FCM/APNs > 디바이스 > 전달 결과 수집
- DIA-032: 캠페인 생성 유저 플로우 (플로우차트) — 메시지 작성 > 오디언스 선택 > 스케줄 설정 > 확인 > 승인 요청 > 발송
- DIA-033: 시스템 아키텍처 (시스템 구성도) — 관리 콘솔 > API 서버 > 잡 큐(스케줄러) > FCM/APNs > 디바이스, 분석 데이터 수집 파이프라인

**검증:** Mermaid 문법 렌더링 확인, PRD 데이터 모델과의 정합성, DIA-GLO-002 상태 머신 패턴과의 일관성
