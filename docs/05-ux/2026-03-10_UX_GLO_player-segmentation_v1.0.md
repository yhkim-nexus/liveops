## 1. Information Architecture

### 1.1 사이트맵 (세그멘테이션 영역)

```
[관리자 대시보드]
└── 세그멘테이션 (Segmentation)
    ├── 오디언스 (SCR-001)                ← 기본 랜딩
    │   ├── 오디언스 목록
    │   ├── 오디언스 생성 (SCR-002)
    │   ├── 오디언스 편집 (SCR-002)
    │   └── 오디언스 상세 (SCR-003)
    ├── 속성 관리 (SCR-004)
    │   ├── Default Properties 목록
    │   ├── Computed Properties 목록
```

#### 기본 정보

| 항목 | 내용 |

| 화면 ID | SCR-001 |
| 화면명 | 오디언스 목록 |
| URL/Route | `/segmentation/audiences` |
| 인증 필요 | Y |
| 접근 권한 | 관리자, 기획자, 운영자 |

#### 화면 목적

오디언스(플레이어 세그먼트) 전체 목록을 조회하고, 검색/필터/정렬로 원하는 오디언스를 빠르게 찾는다. 각 오디언스의 사용 현황(In Use / Unused)을 한눈에 파악하고, 생성·복제·삭제 액션을 수행한다.

### SCR-002: 오디언스 생성/편집

#### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-002 |
| 화면명 | 오디언스 생성/편집 |
| URL/Route | `/segmentation/audiences/new` (생성) / `/segmentation/audiences/:id/edit` (편집) |
| 인증 필요 | Y |
| 접근 권한 | 관리자, 기획자, 운영자 |

#### 화면 목적

오디언스의 기본 정보(이름, 설명)와 필터 조건을 정의한다. 비기술 운영자를 위한 **폼 빌더 뷰**(드롭다운 UI)와 고급 사용자를 위한 **쿼리 뷰**(텍스트 에디터)를 전환하며 사용할 수 있다. 필터 조건에 매칭되는 예상 플레이어 수를 실시간으로 미리보기하고, Include/Exclude 수동 목록을 관리한다.

**Custom 속성 생성**: `POST /api/v1/properties/custom` body: `{ "name": "totalPlayMinutes", "type": "numeric", "description": "총 플레이 시간 (분 단위)" }`

**Custom 속성 수정**: `PUT /api/v1/properties/custom/:name`

**Custom 속성 삭제**: `DELETE /api/v1/properties/custom/:name`
