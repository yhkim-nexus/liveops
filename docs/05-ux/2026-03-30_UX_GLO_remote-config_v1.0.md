# 화면 정의서: Key-Value 원격 설정 (Remote Config)

> Game LiveOps Service(GLO)의 Key-Value 원격 설정(Remote Config) 기능 화면 정의서. 운영팀이 앱 재배포 없이 클라이언트/서버 동작을 실시간으로 제어할 수 있도록 설정 키 CRUD, 타입별 값 편집, 변경 이력 추적 UI를 정의한다. Viewer 역할은 조회만, Editor 이상 역할은 생성·수정·삭제를 수행하며, dot notation 키 체계와 JSON 타입 편집기를 핵심 UI로 제공한다.

## 1. Information Architecture

### 1.1 사이트맵 (Remote Config 영역)

```
[관리자 대시보드]
├── 대시보드 (/dashboard)                   ← Viewer 이상
├── 플레이어 (/players/*)                   ← Viewer 이상
├── 세그먼트 (/segments/*)                  ← Editor 이상
├── 이벤트 (/events/*)                      ← Editor 이상
├── 실험 (/experiments/*)                   ← Editor 이상
├── Remote Config (/remote-config/*)        ← Viewer 이상 (조회), Editor 이상 (편집)
│   ├── 설정 목록 (SCR-033)                 ← /remote-config (Viewer+)
│   ├── 설정 추가 (SCR-034)                 ← /remote-config/new (Editor+)
│   ├── 설정 상세 (SCR-035)                 ← /remote-config/[id] (Viewer+)
│   └── 설정 편집 (SCR-036)                 ← /remote-config/[id]/edit (Editor+)
├── 푸시 알림 (/push/*)                     ← Editor 이상
└── 설정 (/settings/*)                      ← Admin 전용
```

### 1.2 네비게이션 구조

**좌측 사이드바 (Primary Navigation)**

| 순서 | 라벨 | 화면 ID | 경로 |
|------|------|---------|------|
| 1 | 대시보드 | - | /dashboard |
| 2 | 플레이어 | SCR-026 | /players |
| 3 | 세그먼트 | SCR-001 | /segments |
| 4 | 이벤트 | SCR-006 | /events |
| 5 | 실험 | SCR-013 | /experiments |
| **6** | **Remote Config** | **SCR-033** | **/remote-config** |
| 7 | 푸시 알림 | SCR-028 | /push/campaigns |
| 8 | 설정 | - | /settings |

> 사이드바에서 "실험(Experiments)" 아래, "푸시 알림(Push Notifications)" 위에 "Remote Config" 항목을 추가한다.

**Remote Config 서브 네비게이션 없음**

Remote Config는 단일 목록 페이지를 루트로 하는 플랫 구조다. 별도의 Secondary Navigation(서브 탭)은 사용하지 않는다.

### 1.3 화면-기능 매핑

| 화면 ID | 화면명 | 주요 기능 | URL/Route |
|---------|--------|----------|-----------|
| SCR-033 | Remote Config 목록 | 설정 목록 조회, 검색, 필터링, 정렬 | `/remote-config` |
| SCR-034 | Remote Config 추가 | 새 설정 키-값 생성 | `/remote-config/new` |
| SCR-035 | Remote Config 상세 | 설정 조회, 변경 이력 확인 | `/remote-config/[id]` |
| SCR-036 | Remote Config 편집 | 기존 설정 값·메타데이터 수정 | `/remote-config/[id]/edit` |

## 2. 화면 상세

### SCR-033: Remote Config 목록

#### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-033 |
| 화면명 | Remote Config 목록 |
| URL/Route | `/remote-config` |
| 관련 기능 | 설정 목록 조회, 검색, 필터링 |
| 인증 필요 | Y |
| 접근 권한 | Viewer 이상 |

#### 화면 목적

전체 Remote Config 설정 항목을 조회하고, 키 또는 설명 기반 검색과 Target·Type 필터로 원하는 설정을 빠르게 탐색한다. 각 설정의 현재 값, 타입, 적용 대상, 최근 수정 시각을 한눈에 파악하며, Editor 이상 역할은 새 설정을 추가할 수 있다.

**행(Row) 호버 상태**

```
│ game.reward.daily_max  1000  number  client  event +1  2시간 전  [편집]  [삭제]  │
```

> - Viewer 역할: 호버 액션 없음 (행 클릭만 가능)
> - Editor 이상: 호버 시 `[편집]` `[삭제]` 버튼 노출
> - Admin: 호버 시 `[편집]` `[삭제]` 버튼 노출

#### 데이터 요구사항

**API**: `GET /api/v1/remote-config`

```json
{
  "total": 24,
  "page": 1,
  "pageSize": 10,
  "items": [
    {
      "id": "rc_001",
      "key": "game.reward.daily_max",
      "value": "1000",
      "type": "number",
      "target": "client",
      "description": "일일 최대 보상 지급 횟수",
      "tags": ["event", "reward"],
      "updatedAt": "2026-03-30T10:00:00Z",
      "updatedBy": "operator@example.com"
    }
  ]
}
```

## 3. 공통 컴포넌트

### 3.1 타입별 Value 표시 규칙

Remote Config에서 Value는 내부적으로 항상 문자열(string)로 저장되며, Type에 따라 UI에서 파싱·포맷팅하여 표시한다.

| Type | 저장 형식 | 목록 표시 | 상세 표시 | 편집 입력 |
|------|----------|----------|----------|----------|
| string | `"hello"` | `"hello"` (최대 40자 truncate) | 전체 텍스트 | Input (text) |
| number | `"1000"` | `1000` | `1000` | Input (number) |
| boolean | `"true"` / `"false"` | `true` / `false` | `true` + 의미 레이블 | Select (true/false) |
| json | `"{\"key\":\"v\"}"` | `{...}` (고정 표시) | 코드블록 + 문법 강조 | Textarea (monospace) |

### 3.2 TypeBadge 컴포넌트

```
타입별 Badge 스타일 (variant: outline)
- string:  파란 테두리, 파란 텍스트  #3B82F6
- number:  주황 테두리, 주황 텍스트  #F59E0B
- boolean: 초록 테두리, 초록 텍스트  #10B981
- json:    보라 테두리, 보라 텍스트  #8B5CF6
```

### 3.3 TargetBadge 컴포넌트

```
Target별 Badge 스타일 (variant: solid)
- client: 청록 배경, 흰 텍스트   #0891B2
- server: 인디고 배경, 흰 텍스트 #4F46E5
- both:   회색 배경, 흰 텍스트   #6B7280
```

### 3.4 ChangeLogTable 컴포넌트

SCR-035 변경 이력 테이블은 재사용 가능한 `ChangeLogTable` 컴포넌트로 구현한다.

```
Props:
  - configId: string         — 대상 설정 ID
  - pageSize?: number        — 기본값: 10

Columns:
  - 시각: 절대 시각 YYYY-MM-DD HH:mm (KST)
  - 변경자: 이메일 (최대 30자 truncate)
  - 유형: 생성(초록) / 수정(파랑) / 삭제(빨강) Badge
  - 필드: 변경된 필드명 (생성 시 "-")
  - 이전값: 최대 20자 truncate (생성 시 "-")
  - 새값: 최대 20자 truncate
```

### 3.5 ChipInput 컴포넌트

Tags 입력에 사용되는 `ChipInput` 컴포넌트. SCR-034, SCR-036 공통 사용.

```
Props:
  - value: string[]            — 현재 태그 배열
  - onChange: (v: string[]) => void
  - maxItems?: number          — 기본값: 10
  - maxLength?: number         — 개당 최대 글자, 기본값: 32
  - placeholder?: string

동작:
  - 인풋에 텍스트 입력 후 Enter 또는 "추가" 버튼으로 칩 추가
  - 중복 태그 입력 시 인풋 shake 애니메이션 + "이미 추가된 태그입니다." 툴팁
  - maxItems 도달 시 인풋 비활성화 + "최대 N개까지 추가할 수 있습니다." 안내
  - 칩 우측 × 버튼으로 개별 삭제
```

### 3.6 Key 입력 규칙 (dot notation)

```
허용 패턴: ^[a-z0-9._]+$
- 영문 소문자만 허용 (대문자 불허)
- 숫자 허용
- dot(.) 으로 계층 구분: game.reward.daily_max
- underscore(_) 허용: daily_max
- 공백, 하이픈(-), 슬래시(/) 불허

예시 (유효):
  game.reward.daily_max
  ui.theme.primary_color
  server.rate_limit.api_calls
  feature.new_shop.enabled

예시 (무효):
  Game.Reward             — 대문자
  game-reward             — 하이픈
  game reward             — 공백
  .game.reward            — 선행 dot
  game.reward.            — 후행 dot
```

## 5. 접근성 및 반응형

### 5.1 접근성 기준

| 항목 | 기준 | 적용 화면 |
|------|------|----------|
| 터치/클릭 영역 | 최소 44×44px | 모든 버튼, 칩 × 버튼, 페이지네이션 |
| 색상 대비 | 4.5:1 이상 (WCAG AA) | 뱃지 텍스트, 테이블 텍스트, 인라인 오류 메시지 |
| 스크린 리더 | 모든 인터랙티브 요소 aria-label 부여 | 뱃지, 칩 × 버튼, 호버 액션 버튼 |
| 키보드 내비게이션 | Tab 순서 논리적 유지 | SCR-034, SCR-036 폼 전체 |
| 폼 오류 | aria-describedby로 오류 메시지 연결 | 모든 인풋 필드 |
| 읽기 전용 필드 | aria-readonly="true" + 시각적 비활성 표시 | E-036-01 Key 인풋 |
| 로딩 상태 | aria-busy="true" + 스크린 리더용 "불러오는 중" 메시지 | 테이블 로딩, 저장 버튼 로딩 |

### 5.2 반응형 레이아웃

현재 관리자 대시보드는 데스크탑 브라우저(최소 1280px)를 기준으로 한다.

| 브레이크포인트 | 적용 |
|--------------|------|
| ≥ 1280px | 기본 레이아웃 (좌측 사이드바 240px + 컨텐츠 영역) |
| 1024px ~ 1279px | 사이드바 축소 (아이콘만 표시, 64px) |
| < 1024px | 미지원 (모바일 관리자 화면은 v2 범위) |

**SCR-033 목록 테이블 컬럼 우선순위 (좁은 화면 순)**

| 우선순위 | 컬럼 | 최소 폭 조건 |
|---------|------|------------|
| 1 (항상 표시) | Key | 모든 해상도 |
| 2 | Value | ≥ 1024px |
| 3 | Type 뱃지 | ≥ 1024px |
| 4 | Target 뱃지 | ≥ 1280px |
| 5 | Tags | ≥ 1280px |
| 6 | Updated | ≥ 1280px |
