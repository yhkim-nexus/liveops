# 화면 정의서: A/B 테스트(실험)

> Game LiveOps Service(GLO)의 A/B 테스트(실험) 기능 화면 정의서. 실험 생성부터 승자 롤아웃까지 전 주기를 비기술 직군 운영자가 운영할 수 있도록 5단계 위저드와 통계 결과 대시보드를 제공한다.

## 1. Information Architecture

### 1.1 사이트맵 (실험 영역)

```
[관리자 대시보드]
└── 실험 (Experiments)
    ├── 실험 목록 (SCR-013)              ← 기본 랜딩

## 2. SCR-013: 실험 목록

#### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-013 |
| URL/Route | `/experiments` |
| 인증 필요 | Y |
| 접근 권한 | experiment_creator, experiment_approver, experiment_operator |

#### 화면 목적

전체 실험 목록을 상태·기간 필터와 검색으로 빠르게 탐색한다. 실험의 현재 상태, 오디언스, 변형 수, 기간을 한눈에 파악하고 생성·보관 액션을 수행한다.

#### 와이어프레임 (Default State)

```
┌────────────────────────────────────────────────────────────────┐
│  [GLO]  Overview  Segmentation  Live Events  [Experiments]  …  │  GNB
├────────────────────────────────────────────────────────────────┤
│  실험 (Experiments)                                             │  Page Title
│  [실험 목록]  [승인 대기 (2)]                                   │  Sub Tab
├────────────────────────────────────────────────────────────────┤
│  실험 목록                              [+ 새 실험 만들기]       │
│  [실험명 검색...] 🔍  │ 상태 ▼ │ 정렬: 생성일 ▼               │
│  전체 15개                                                      │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ 실험명          상태         오디언스    변형  시작일  종료일 │  │
│  ├──────────────────────────────────────────────────────────┤  │
│  │ 보상량 A/B     [● running]  활성 유저   2+C   03/20  04/20 │  │
│  │ 신규 온보딩 UI  [● testing]  신규 유저   1+C   –      04/10 │  │
│  │ 난이도 실험     [● draft]   VIP 오디언스 2+C   –      –    │  │
│  │ 시즌 UI        [● completed] 전체       1+C   02/01  03/01 │  │
│  └──────────────────────────────────────────────────────────┘  │
│  ← 이전   1  2   다음 →                                         │
└────────────────────────────────────────────────────────────────┘
```

> 행 호버 시 상태에 따라 액션 버튼 표시:
> - `draft`: `[편집] [복제] [삭제]`
> - `testing/pending_approval/running/paused`: `[상세] [복제] [보관]`
> - `completed/archived`: `[상세] [복제]`

#### 상태 변형

- **Loading**: 테이블 영역 스켈레톤 (5행 × 전체 컬럼)

- **Empty (전체 없음)**: "아직 실험이 없습니다." + 새 실험 만들기 CTA
- **Error**: "일시적 오류가 발생했습니다." + 다시 시도 버튼

## 3. SCR-014: 실험 생성/편집 위저드

#### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-014 |
| URL/Route | `/experiments/new`, `/experiments/:id/edit` |
| 인증 필요 | Y |
| 접근 권한 | experiment_creator |
| 편집 가능 상태 | `draft`만 편집 가능. testing 이상 상태에서 /edit 접근 시 SCR-015 리다이렉트 |

#### 화면 목적

5단계 스텝 위저드로 실험을 설정한다. 각 단계 완료 시 자동 저장(draft)되며, 마지막 단계에서 임시 저장 또는 사전 테스트 시작을 선택한다.

#### 와이어프레임 (공통 레이아웃)

```
┌────────────────────────────────────────────────────────────────┐
│  [←] 실험 생성                                                  │  Header
├───────────────┬────────────────────────────────────────────────┤
│  1 기본 정보  │                                                  │
│  2 변형 정의  │   [단계별 폼 영역]                                  │  Content
│  3 오디언스   │                                                 │
│  4 목표 지표  │                                                 │
│  5 확인       │                                                 │
├───────────────┴────────────────────────────────────────────────┤
│                              [이전]  [다음 단계로] / [완료]     │  Footer
└────────────────────────────────────────────────────────────────┘
```

> 좌측 스텝 인디케이터: 완료 단계는 체크(✓), 현재 단계는 강조, 미완료는 비활성.

- Key-Value: 동일 키 중복 입력 → "중복된 키입니다" 인라인 에러

#### 충돌 경고 배너 내용

```
⚠ 동일 오디언스에 실행 중인 실험이 있습니다.
  '보상량 A/B 테스트' 실험이 동일 오디언스에서 실행 중입니다.
  독립 할당 방식으로 진행되며, 결과 해석 시 교호작용 가능성을 고려하세요.

## 7. 공통 컴포넌트

### 7.1 실험 상태 배지 (ExperimentStatusBadge)

- UX-GLO-002의 이벤트 상태 배지와 동일 패턴. 색상 정의는 섹션 2 참조.
- Props: `status: ExperimentStatus`, `size: 'sm' | 'md'`

### 7.2 트래픽 슬라이더 (TrafficSlider)

- 다중 변형 트래픽 배분을 단일 슬라이더 그룹으로 표현
- 한 변형 조정 시 나머지 변형 균등 재분배 (Control 포함)
- 합계 != 100%일 때 빨간색 경고 뱃지 표시
- Props: `variants: {id, name, percentage}[]`, `onChange: (variants) => void`

### 7.3 통계 결과 카드 (StatResultCard)

- 변형명, 지표값, 컨트롤 대비 차이(%), p-value, 95% CI를 카드 형태로 표시
- 승자 변형: 초록 테두리 + 별(★) 뱃지
- "데이터 부족" 상태: 회색 + 수집 진행률 표시
- Props: `variant`, `isWinner`, `isControl`, `result: ExperimentResult`

### 7.4 Key-Value 에디터 (KeyValueEditor)

- 인라인 테이블 형태. 행 추가/삭제 가능
- Key 입력: 기존 원격 설정 키 자동완성 드롭다운 (API: `GET /api/v1/remote-config/keys`)
- Value 입력: 텍스트. JSON 복합 설정 지원 (JSON 유효성 검사 포함)
- 중복 키 감지 → 인라인 에러 표시
- Props: `value: {key, value}[]`, `onChange`, `readOnly?: boolean`
