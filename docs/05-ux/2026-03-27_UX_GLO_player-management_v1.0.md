# 화면 정의서: 플레이어 상태 조회 및 계정 조치

> Game LiveOps Service(GLO)의 플레이어 상태 조회 및 계정 조치 기능에 대한 화면 정의서. 통합 검색으로 플레이어를 조회하고, 상세 정보 확인 및 차단/해제/닉네임 변경 등 계정 조치를 수행하는 화면을 정의한다.

## 1. Information Architecture

### 1.1 사이트맵 (플레이어 관리 영역)

```
[관리자 대시보드]
├── 대시보드 (/dashboard)                ← viewer 이상
├── 이벤트 (/events/*)                   ← operator 이상
├── 실험 (/experiments/*)                ← operator 이상
├── 세그먼트 (/segments/*)               ← operator 이상
├── 플레이어 (Players)                    ← Viewer 이상 (조회), Operator 이상 (계정 조치)
│   ├── 플레이어 목록/검색 (SCR-026)      ← /players (Viewer+)
│   └── 플레이어 상세 (SCR-027)           ← /players/[id] (Viewer+ 조회, Operator+ 조치)
└── 설정 (/settings/*)                   ← admin 전용
```

### 1.2 네비게이션 흐름

```
플레이어 목록 (SCR-026)
  │
  ├── 검색 → 결과 테이블 → 행 클릭 → 플레이어 상세 (SCR-027)
  │                                       │
  │                                       ├── 차단 모달
  │                                       ├── 차단 해제
  │                                       └── 닉네임 변경 모달
  │
  └── ← 뒤로가기 ← 플레이어 상세 (SCR-027)
```

### 1.3 화면-기능 매핑

| 화면 ID | 화면명 | 주요 기능 |
|---------|--------|----------|
| SCR-026 | 플레이어 목록/검색 | 통합 검색(ID/이메일/닉네임), 결과 테이블, 페이지네이션 |
| SCR-027 | 플레이어 상세 | 프로필 조회, 오디언스 확인, 차단/해제, 닉네임 변경 |

### SCR-026: 플레이어 목록/검색

#### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-026 |
| 화면명 | 플레이어 목록/검색 |
| URL/Route | `/players` |
| 인증 필요 | Y |
| 접근 권한 | Viewer 이상 (Viewer/Editor/Operator/Admin) |

## 3. 접근성 기준

- 모든 인터랙티브 요소의 터치/클릭 영역: 최소 44x44px
- 색상 대비: WCAG 2.1 AA 기준 (4.5:1 이상)
- 상태 배지: 색상 외 텍스트 라벨로 상태 구분 가능 (예: "활성", "차단됨", "정지됨")
- 스크린 리더: 테이블 헤더에 scope 속성, 버튼에 aria-label 제공
- 키보드 네비게이션: Tab으로 모든 인터랙티브 요소 접근 가능
- 포커스 표시: 키보드 포커스 시 visible outline 제공
- 모달 다이얼로그: 열릴 때 포커스 트랩 적용, ESC로 닫기 지원
