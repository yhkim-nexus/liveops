# 화면 정의서: 모바일 푸시 알림 발송

> Game LiveOps Service(GLO)의 모바일 푸시 알림 발송 기능 화면 정의서. 운영팀이 관리 콘솔에서 타겟 오디언스에 캠페인성 푸시 알림을 직접 작성·발송·분석할 수 있는 캠페인 관리 UI를 정의한다. 비기술 직군 운영자를 주 대상으로 하며, 5단계 생성 위저드·실시간 디바이스 미리보기·발송 퍼널 분석 대시보드를 핵심 UI로 제공한다.

## 1. Information Architecture

### 1.1 사이트맵 (푸시 알림 영역)

```
[관리자 대시보드]
├── 대시보드 (/dashboard)                ← Viewer 이상
├── 이벤트 (/events/*)                   ← Operator 이상
├── 실험 (/experiments/*)                ← Operator 이상
├── 세그먼트 (/segments/*)               ← Operator 이상
├── 플레이어 (/players/*)                ← Viewer 이상
├── 푸시 알림 (Push Notifications)       ← Editor 이상 (작성), Operator 이상 (승인)
│   ├── 캠페인 목록 (SCR-028)            ← /push/campaigns (Viewer+)
│   │   ├── 캠페인 생성 (SCR-029, /new)
│   │   └── 캠페인 편집 (SCR-029, /[id]/edit)
│   ├── 캠페인 상세/분석 (SCR-030)        ← /push/campaigns/[id]
│   ├── 템플릿 목록 (SCR-031)            ← /push/templates (Viewer+)
│   │   ├── 템플릿 생성 (SCR-032, /new)
│   │   └── 템플릿 편집 (SCR-032, /[id]/edit)
└── 설정 (/settings/*)                   ← Admin 전용
```

#### 화면 목적

전체 푸시 알림 캠페인을 조회하고 상태·날짜 범위 필터와 검색으로 원하는 캠페인을 빠르게 탐색한다. 각 캠페인의 상태, 예약 시간, 발송/오픈 통계를 한눈에 파악하고, Editor 이상 역할은 새 캠페인을 생성할 수 있다.

## 5. 접근성 기준

- 모든 인터랙티브 요소의 클릭 영역: 최소 44×44px
- 색상 대비: WCAG 2.1 AA 기준 (4.5:1 이상)
- 상태 배지: 색상 외 텍스트 라벨로 상태 구분 가능 (예: "발송 완료", "승인 대기")
- 스크린 리더: 테이블 헤더에 `scope` 속성, 버튼에 `aria-label` 제공
- 키보드 네비게이션: Tab으로 모든 인터랙티브 요소 접근 가능
- 위저드 스텝 인디케이터: `aria-current="step"` 속성으로 현재 단계 표시
- 미리보기 패널: `aria-live="polite"` 적용하여 스크린 리더에 변경 사항 알림
- 모달 다이얼로그: 열릴 때 포커스 트랩 적용, ESC로 닫기 지원
