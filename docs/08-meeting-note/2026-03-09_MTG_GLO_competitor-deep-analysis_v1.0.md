## 1. 회의 목적

1차 시장조사(RES-GLO-001)에서 식별된 14개 서비스 중 7개 주요 플랫폼을 선정하여 공식 문서 기반 심층 기능 분석을 수행하고, 자체 LiveOps 서비스의 핵심 기능 범위와 기획 방향을 도출한다.

---

## 2. 논의 내용

### 2.1 레퍼런스 기반 조사 배경

**배경**
- 1차 시장조사(RES-GLO-001)에서 14개 LiveOps 서비스를 식별하고 개략적 비교를 완료
- 사용자가 심층 분석 대상으로 7개 플랫폼을 선정: Heroic Labs Satori, Unity UGS, AccelByte, Hive Console, Balancy, GameAnalytics, Metaplay

**논의 내용**
- 각 플랫폼의 공식 문서(Documentation)를 직접 분석하여 실제 제공 기능을 검증
- 마케팅 자료가 아닌 기술 문서 기반 분석으로 정확도 확보
- 총 13개 출처(공식 문서 페이지)를 참조하여 기능 매핑 수행

**결론**
- 공식 문서 기반 분석으로 1차 조사 대비 기능 정보의 정확도와 깊이가 크게 향상됨

### 2.2 8개 공통 핵심 기능 카테고리 도출

**배경**
- 7개 플랫폼의 기능을 교차 비교하여 공통 카테고리를 도출할 필요성 제기

**논의 내용**
- 분석 결과 LiveOps 시스템은 아래 8개 핵심 카테고리로 수렴:

| No. | 카테고리 | 7개 플랫폼 공통 지원 |
|-----|----------|-------------------|
| 1 | 플레이어 세그멘테이션 | 7/7 |
| 2 | 라이브 이벤트 관리 | 6/7 |
| 3 | 실험 및 A/B 테스트 | 7/7 (부분 포함) |
| 4 | 원격 설정 및 Feature Flags | 7/7 |
| 5 | 푸시 알림 및 메시징 | 6/7 |
| 6 | 분석 및 대시보드 | 7/7 |
| 7 | 게임 경제 시스템 | 5/7 |
| 8 | 플레이어 지원 및 운영 | 5/7 |

- 세그멘테이션 + A/B 테스트 + 원격 설정의 삼각 조합이 모든 플랫폼에서 핵심 기능군으로 확인됨
- 게임 경제(상점, 시즌 패스, 가상 화폐)는 중대형 플랫폼의 필수 구성 요소로 자리잡음

**결론**
- 8개 카테고리를 기능 범위 정의의 기준 프레임워크로 채택

### 2.3 필수/권장/선택 기능 우선순위 분류

**배경**
- 8개 카테고리 내 세부 기능을 우선순위별로 분류하여 MVP 범위 확정에 활용

**논의 내용**
- **필수 기능 (Must-Have) 8개**: 세그먼트 관리, 원격 설정, 이벤트 스케줄링, A/B 테스트, 푸시 알림, 핵심 대시보드, 플레이어 상태 조회, RBAC
- **권장 기능 (Should-Have) 10개**: Feature Flag, 타겟 푸시, 이벤트 모니터링, 가상 화폐/상점, 인게임 메일, GDPR 도구, 이벤트 캘린더, 감사 로그, 퍼널/리텐션 분석 등
- **선택 기능 (Nice-to-Have) 10개**: 시즌/배틀 패스, 멀티 페이즈 A/B, 데이터 웨어하우스 연동, Visual Scripting, 쿠폰, 길드 관리, 코호트 분석 등

**결론**
- 3단계 우선순위 분류를 Phase별 기획의 근거 자료로 활용

### 2.4 Phase별 기획 방향 제안

**배경**
- 필수/권장/선택 분류를 기반으로 단계적 출시 전략 수립

**논의 내용**
- **Phase 1 (MVP)**: 세그멘테이션 + 원격 설정 + 이벤트 스케줄링 + 기본 대시보드 + 푸시 알림 (5개 핵심 기능)
- **Phase 2**: A/B 테스트 + 인게임 메일 + 경제 시스템(상점/인벤토리/화폐) + GDPR 도구
- **Phase 3**: Feature Flag 고도화 + 시즌/배틀 패스 + 데이터 파이프라인 연동 + Config Diff 도구
- 기회 영역 3가지 식별:
  1. 운영자 친화적 통합 인터페이스 (대부분 개발자 중심 UI)
  2. 세그먼트-이벤트-보상 통합 파이프라인
  3. 실시간 이벤트 모니터링 + 즉각 개입 기능

**결론**
- 3-Phase 로드맵을 PRD 작성의 기초 자료로 활용
- 운영자 중심 UX를 핵심 차별화 전략으로 확정
