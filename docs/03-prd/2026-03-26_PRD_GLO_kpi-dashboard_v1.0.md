## 1. Overview

### 1.1 제품 비전

LiveOps 운영팀이 DAU, 리텐션, 수익 핵심 지표를 한 화면에서 일간 기준으로 모니터링할 수 있는 스냅샷 대시보드를 제공한다. KPI Summary 카드 4개와 3개 영역(회원 활동, 리텐션, 수익) 트렌드 차트를 단일 인터페이스에서 확인하여, 비기술 운영자가 데이터 기반 의사결정을 즉시 수행할 수 있다.

### 1.2 배경 및 목적

**배경**

- 경쟁사 분석(RES-GLO-002, Section 3.6)에서 "핵심 지표 대시보드 (DAU, 리텐션, 수익)"는 Must-Have #6으로 분류되었다. 7개 경쟁 플랫폼 모두 실시간 플레이어 지표를 제공하며, 커스텀 대시보드(5/7), 퍼널 분석(4/7), 코호트 분석(3/7) 순으로 지원한다.
- 리서치 권고사항 #2: "관리자 대시보드는 기술자 전용이 아닌 운영자(LiveOps팀) 중심의 인터페이스로 설계할 것" — 이를 반영하여 비기술 운영자가 별도 설정 없이 즉시 핵심 지표를 파악할 수 있는 사전 구축(pre-built) 대시보드를 설계한다.
- GameAnalytics의 사전 구축 대시보드와 Heroic Labs Satori의 커스텀 위젯 대시보드 모델을 벤치마킹한다.
- PLO Phase 1~3(세그멘테이션, 라이브 이벤트, A/B 테스트)에서 수집된 회원 활동, 리텐션, 수익 데이터를 종합 시각화하는 모니터링 허브 역할을 수행한다.

**목적**

- 이 PRD는 GLO 서비스의 **핵심 지표 대시보드(KPI Dashboard)** 기능 범위를 정의한다.
- KPI Summary 카드, 회원 활동 트렌드 차트, 리텐션 커브 차트, 수익 트렌드 차트를 명세한다.
- MVP(Phase 1) 출시에 필요한 최소 기능과 수용 기준을 확정한다.

### 1.3 성공 지표 (Success Metrics)

| 지표 | 현재 | 목표 | 측정 방법 |
|------|------|------|----------|
| 페이지 로드 시간 | - | 2초 이내 | 대시보드 진입 ~ 모든 차트 렌더링 완료까지 소요 시간 |
| 기간 토글 전환 시간 | - | 500ms 이내 | 기간 버튼 클릭 ~ 차트 리렌더링 완료까지 소요 시간 |
| 일일 대시보드 확인율 | - | 80% 이상 | 전체 관리자 중 대시보드 페이지를 1일 1회 이상 방문한 비율 |
| 데이터 정확도 | - | 99.9% | 대시보드 표시 수치와 원본 데이터 간 일치율 |

## 2. Target Users

### 2.1 사용자 페르소나

#### Primary Persona: 김지연 (게임 기획자/LiveOps 운영자)

PRD-GLO-001에서 정의된 동일 페르소나를 재사용한다.

| 항목 | 내용 |
|------|------|
| 소속 | 인디~중소 게임사 (팀 규모 5~30명) |
| 역할 | 게임 기획자 겸 LiveOps 운영 담당 |
| 기술 수준 | 비개발자. SQL 기초 이해 가능, 코드 작성 불가 |
| 주요 도구 | 스프레드시트, Notion, 간단한 BI 대시보드 |

**Goals**
- 매일 아침 대시보드에 접속해 DAU, 리텐션, 매출 핵심 지표를 한눈에 확인하고 싶다.
- 전일 대비 변화율을 색상과 아이콘으로 직관적으로 파악하고 싶다.
- 7일/30일/90일 기간별 트렌드를 빠르게 전환하며 추세를 분석하고 싶다.

**Pain Points**
- 여러 도구를 오가며 DAU, 리텐션, 매출을 각각 확인해야 한다.
- 스프레드시트 기반 지표 관리는 수동 업데이트가 필요해 최신 데이터 확인이 어렵다.
- 비기술 운영자가 BI 도구의 복잡한 쿼리를 작성하지 않고도 핵심 지표를 볼 수 있는 방법이 없다.

#### Secondary Persona: 박현수 (개발자/기술 운영자)

PRD-GLO-001에서 정의된 동일 페르소나를 재사용한다.

| 항목 | 내용 |
|------|------|
| 소속 | 중소 게임사 서버 개발자 |
| 역할 | LiveOps 백엔드 연동 및 기술 성능 모니터링 |
| 기술 수준 | SQL, 프로그래밍 언어 숙련 |

**Goals**
- 서비스 성능 이상 징후(DAU 급락, 리텐션 급변)를 대시보드에서 빠르게 감지하고 싶다.
- 배포 후 핵심 지표 변화를 확인하여 릴리스 영향도를 판단하고 싶다.

**Pain Points**
- 서버 로그와 별도로 비즈니스 지표를 확인하려면 운영팀에 요청해야 한다.
- 긴급 이슈 발생 시 지표 변화 추세를 빠르게 파악할 단일 화면이 없다.

### 2.2 사용자 시나리오

#### SC-010: 일일 KPI 모니터링

> 김지연은 매일 아침 대시보드에 접속하여 핵심 지표를 확인하고, 전일 대비 변화를 파악하여 당일 운영 계획을 수립한다.

**Steps**
1. 김지연이 관리자 대시보드 메인(SCR-KPI-001)에 접속한다
2. KPI Summary 카드 4개(DAU, MAU, D1 리텐션, 일 매출)에서 현재 수치와 전일/전월 대비 변화율을 확인한다
3. DAU 변화율이 빨간색(▼)으로 표시됨을 발견 → 회원 활동 트렌드 차트에서 7D 기간을 선택하여 최근 추세를 확인한다
4. DAU 하락이 주말 패턴임을 확인 → 30D로 전환하여 전체 트렌드가 상승세임을 확인한다
5. 리텐션 커브 차트에서 최신 코호트와 이전 코호트를 비교하여 D1/D7 리텐션 개선 여부를 확인한다
6. 수익 트렌드 차트에서 일 매출, ARPU, ARPPU 추이와 결제 전환율을 확인한다
7. 전체 지표를 종합하여 당일 운영 의사결정(이벤트 연장, 프로모션 조정 등)을 판단한다

#### SC-011: 이벤트 후 지표 변화 확인

> 박현수는 라이브 이벤트 종료 후 대시보드에서 이벤트 기간 동안의 매출 및 DAU 변화를 확인한다.

**Steps**
1. 박현수가 대시보드에 접속하여 수익 트렌드 차트를 90D 기간으로 설정한다
2. 이벤트 기간(약 7일간)의 일 매출 Bar 높이가 전후 대비 증가했음을 확인한다
3. ARPU와 ARPPU 라인 추이를 통해 결제자 1인당 매출 변화를 분석한다
4. 회원 활동 트렌드 차트에서 이벤트 기간 동안 DAU와 세션 수 증가를 확인한다
5. 분석 결과를 운영팀에 공유하여 다음 이벤트 기획에 반영한다

### 3.2 기능 상세: F-024 KPI Summary Cards

#### 개요

DAU, MAU, D1 리텐션, 일 매출 4개 핵심 지표를 카드 형태로 대시보드 상단에 배치한다. 각 카드는 현재 값, 비교 기준값 대비 변화율, 색상 코딩(초록/빨강/회색)을 포함하여 운영자가 즉시 서비스 상태를 파악할 수 있다.

#### 사용자 스토리

**US-036**
```
As a 게임 기획자(관리자),
I want to DAU, MAU, D1 리텐션, 일 매출 핵심 지표를 한 줄에 4개 카드로 확인하기를,
So that 대시보드 접속 즉시 서비스 전반의 건강 상태를 파악할 수 있다.
```

**US-037**
```
As a 게임 기획자(관리자),
I want to 각 KPI 카드에서 전일/전월 대비 변화율을 색상과 아이콘으로 구분하기를,
So that 지표의 개선/악화를 직관적으로 인지하고 즉각 대응할 수 있다.
```

#### 상세 요구사항

| ID | 요구사항 | 필수 |
|----|----------|------|
| REQ-024-01 | KPI Summary 영역은 DAU, MAU, D1 리텐션, 일 매출 4개 카드로 구성되며, 상단에 4열 균등 배치한다. | Y |
| REQ-024-02 | 각 카드는 지표명(한글 레이블 + 영문 약어), 현재 값, 변화율, 비교 기준값을 표시한다. | Y |
| REQ-024-03 | DAU 카드는 "일간 활성 회원(DAU)"으로 표시하며, 전일 대비 변화율(%)을 제공한다. 값은 K/M 단위로 축약한다. | Y |
| REQ-024-04 | MAU 카드는 "월간 활성 회원(MAU)"으로 표시하며, 전월 대비 변화율(%)을 제공한다. 값은 K/M 단위로 축약한다. | Y |
| REQ-024-05 | D1 리텐션 카드는 "어제 가입자 오늘 복귀율"로 표시하며, 전일 대비 변화를 퍼센트 포인트(pp) 단위로 제공한다. | Y |
| REQ-024-06 | 일 매출 카드는 "오늘 총 매출"로 표시하며, 전일 대비 변화율(%)을 제공한다. 값은 통화($) K/M 단위로 축약한다. | Y |
| REQ-024-07 | 양수 변화: 초록색(text-green-600), ▲ 아이콘. 음수 변화: 빨간색(text-red-600), ▼ 아이콘. 변화 없음: 회색(text-gray-500), — 표시. | Y |
| REQ-024-08 | API 엔드포인트 `GET /api/dashboard/summary`에서 데이터를 조회하며, 쿼리 파라미터 `date`(YYYY-MM-DD, 기본: 오늘)를 지원한다. | Y |
| REQ-024-09 | API 응답은 `DashboardSummary` 타입을 따르며, 각 지표는 `KpiMetric` 인터페이스(value, unit, changeRate, comparisonValue)로 구성된다. | Y |

#### 수용 조건 (Acceptance Criteria)

**AC-024-01**
```gherkin
Scenario: KPI Summary 카드 정상 표시
  Given 관리자가 KPI 대시보드(SCR-KPI-001)에 접속한다
  When 페이지가 로드된다
  Then DAU, MAU, D1 리텐션, 일 매출 4개 카드가 상단에 4열 균등 배치로 표시된다
  And 각 카드에 현재 값과 변화율이 표시된다
```

**AC-024-02**
```gherkin
Scenario: 양수 변화율 색상 표시
  Given DAU가 전일 대비 5.2% 증가했다
  When KPI Summary 카드가 렌더링된다
  Then DAU 카드의 변화율이 "▲+5.2%"로 표시된다
  And 변화율 텍스트 색상이 초록색(text-green-600)이다
```

**AC-024-03**
```gherkin
Scenario: 음수 변화율 색상 표시
  Given D1 리텐션이 전일 대비 1.2pp 감소했다
  When KPI Summary 카드가 렌더링된다
  Then D1 리텐션 카드의 변화율이 "▼-1.2pp"로 표시된다
  And 변화율 텍스트 색상이 빨간색(text-red-600)이다
```

**AC-024-04**
```gherkin
Scenario: 변화 없음 표시
  Given MAU가 전월과 동일하다 (changeRate = 0)
  When KPI Summary 카드가 렌더링된다
  Then MAU 카드의 변화율이 "— 0%"로 표시된다
  And 변화율 텍스트 색상이 회색(text-gray-500)이다
```

## 6. Dependencies & Constraints

### 6.1 의존성

| 결제 데이터 집계 | 일 매출, 결제건수, ARPU, ARPPU 계산 | GLO 팀 | P0 | Mock 데이터 기반 |
| GLO SDK 이벤트 수집 | 플레이어 활동 이벤트 수신 | GLO 팀 | P0 | 개발 중 |

### 6.2 제약사항

**기술적 제약**
- MVP는 Mock 데이터 기반으로 구현한다. 실제 데이터 파이프라인 연동은 후속 버전에서 진행한다.
- 4개 API 엔드포인트는 Next.js API Route로 구현하며, Mock 데이터를 반환한다.
- 차트 라이브러리는 Recharts를 사용한다. 기존 프로젝트 기술 스택과 일치해야 한다.

**일정 제약**
- 세그먼트별 필터링, 임계값 알림, 데이터 Export는 MVP 범위에서 제외한다.

**리소스 제약**
- 실시간 스트리밍 데이터, 커스텀 대시보드 위젯은 MVP 범위에서 제외한다.

### 6.3 가정사항

- 대시보드 데이터는 일간 배치 집계 기반이며, 실시간 스트리밍이 아니다.
- MVP에서는 전체 플레이어 기준 지표만 제공하며, 세그먼트별 필터링은 v1.1에서 지원한다.
- Mock API의 응답 형식은 실제 API 연동 시에도 동일하게 유지한다.

## Appendix

### A. 용어 정의

| 용어 | 정의 |
|------|------|
| DAU (Daily Active Users) | 일간 활성 회원. 해당일에 1회 이상 세션을 생성한 고유 회원 수 |
| MAU (Monthly Active Users) | 월간 활성 회원. 해당월에 1회 이상 세션을 생성한 고유 회원 수 |
| D1 리텐션 | 가입일 다음날 복귀한 회원 비율. (D1 복귀자 수 / 가입자 수) × 100 |
| ARPU (Average Revenue Per User) | 회원당 평균 수익. 일 매출 / DAU |
| ARPPU (Average Revenue Per Paying User) | 결제 회원당 평균 수익. 일 매출 / 결제자 수 |
| 코호트 (Cohort) | 동일 시점에 가입한 회원 그룹. 리텐션 분석의 기본 단위 |
| 결제 전환율 | 전체 활성 회원 중 결제를 수행한 회원의 비율 |
| 기간 토글 | 7D/30D/90D 기간 선택 UI. 선택에 따라 차트의 X축 범위가 변경됨 |
| KPI (Key Performance Indicator) | 핵심 성과 지표. 서비스 운영 목표 달성도를 측정하는 정량 지표 |
| 스냅샷 대시보드 | 사전 구축된 고정 레이아웃 대시보드. 커스텀 위젯 편집 없이 핵심 지표를 즉시 확인 가능 |
